**Labels:** `documentation`, `low-priority`

## Problem
No contribution guidelines exist for external contributors.

## Impact
- Inconsistent contributions
- More review iterations needed
- Unclear process for contributors

## Proposed Solution
Create `CONTRIBUTING.md` with:

### 1. Code of Conduct
Basic respectful behavior guidelines

### 2. How to Contribute
- Fork the repository
- Create feature branch
- Make changes
- Run tests
- Submit PR

### 3. Development Setup
- Prerequisites
- Installation steps
- Running locally
- Running tests

### 4. Code Style
- Follow ESLint rules
- Use Prettier formatting
- TypeScript required
- Component patterns

### 5. Commit Messages
Follow Conventional Commits:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `test:` Tests
- `refactor:` Code refactoring
- `chore:` Maintenance

### 6. Pull Request Process
- Describe changes clearly
- Reference related issues
- All tests must pass
- Update documentation
- Request review

### 7. Testing Requirements
- Unit tests for new logic
- E2E tests for new features
- No breaking changes without discussion

## Success Criteria
- Clear contribution process
- Reduced friction for contributors
- Consistent code quality
- Faster review cycles
